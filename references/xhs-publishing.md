# 发布与复盘

笔记打磨完成后，同步到小红书/抖音草稿箱，发布后追踪数据表现。

## 同步发布

通过 Wechatsync 同步到小红书和抖音图文的草稿箱。

### 前置条件

1. **安装 Wechatsync CLI**：
   ```bash
   npm install -g @wechatsync/cli
   ```

2. **安装 Chrome 扩展**：在 Chrome 网上应用店搜索「文章同步助手」

3. **登录目标平台**：在浏览器里正常登录小红书/抖音（使用浏览器 Cookie）

4. **启用 MCP 连接**：在扩展设置中启用并获取 Token

5. **设置环境变量**：
   ```bash
   export WECHATSYNC_TOKEN="你的token"
   ```

### 检查登录状态

```bash
wechatsync platforms --auth
```

### 同步命令

```bash
# 同步到小红书草稿箱
wechatsync sync output/xxx.md -p xiaohongshu -t "标题"

# 同步到抖音图文草稿箱
wechatsync sync output/xxx.md -p douyin -t "标题"

# 同时同步到两个平台
wechatsync sync output/xxx.md -p xiaohongshu,douyin -t "标题"
```

**可选参数：**
- `-t "标题"` — 指定标题（默认从文件提取）
- `--cover URL` — 指定封面图
- `--dry-run` — 预览模式，不同步

### 同步后流程

1. AI 提示：「已同步到小红书草稿箱，请在 APP 中检查排版」
2. 用户去小红书 APP 检查
3. 确认排版、图片是否正常
4. 用户手动点击发布

### 平台差异

| 差异点 | 小红书 | 抖音图文 |
|--------|--------|----------|
| 标题长度 | 20 字以内 | 55 字以内 |
| 封面比例 | 3:4 竖图 | 9:16 竖图 |
| 正文长度 | 300-800 字 | 300-1000 字 |
| 标签数量 | 5-10 个 | 3-5 个 |
| 风格 | 种草、真实感 | 更视觉化、更短平快 |

**建议**：生成两个版本的内容包，分别适配两个平台。

## 数据追踪

发布后记录数据表现，分析经验教训。

### 数据指标

**基础数据（必追）：**

| 指标 | 说明 | 重要性 |
|------|------|--------|
| 浏览量 | 笔记被多少人看到 | ⭐⭐⭐⭐⭐ |
| 点赞数 | 用户认可内容 | ⭐⭐⭐⭐⭐ |
| 收藏数 | 内容有实用价值 | ⭐⭐⭐⭐⭐ |
| 评论数 | 引发讨论/互动 | ⭐⭐⭐⭐⭐ |
| 分享数 | 社交裂变能力 | ⭐⭐⭐⭐ |
| 涨粉数 | 笔记带来的新粉丝 | ⭐⭐⭐⭐ |

**进阶指标（建议追）：**

| 指标 | 说明 | 为什么重要 |
|------|------|-----------|
| 点击率 | 曝光→点击的比例 | 衡量封面+标题吸引力 |
| 互动率 | (点赞+收藏+评论)/浏览 | 衡量内容质量 |
| 收藏率 | 收藏/浏览 | 干货笔记核心指标 |
| 评论质量 | 是否有深度评论 | 衡量内容触动程度 |

**归因分析（用于优化爆款公式）：**

| 指标 | 说明 | 优化方向 |
|------|------|---------|
| 封面类型 | 对比图/大字报/场景图 | 哪种封面点击率高 |
| 标题公式 | 数字型/痛点型/悬念型 | 哪种标题更吸引人 |
| 发布时间 | 7-9点/12-14点/20-21点 | 哪个时段效果好 |
| 标签效果 | 哪些标签带来了流量 | 优化标签策略 |

### 记录时间点

| 时间点 | 记录什么 |
|--------|---------|
| 1 小时 | 初步数据，判断是否限流 |
| 24 小时 | 核心数据，判断爆款潜力 |
| 72 小时 | 稳定数据，判断长尾效果 |
| 7 天 | 最终数据，总结经验 |

## 复盘

数据记录完成后，分析成功/失败因素，更新爆款公式。

### 复盘内容

**成功因素分析：**
- 哪些元素表现好？
- 为什么能吸引用户？
- 哪些标签带来了流量？
- 哪个时间段效果最好？

**失败因素分析：**
- 哪些元素表现差？
- 为什么没有吸引用户？
- 哪些标签没有效果？
- 是否被限流？为什么？

**改进方向：**
- 下次可以改进什么？
- 哪些元素可以复用？
- 哪些元素需要调整？

### 复盘报告格式

```markdown
## 复盘笔记

### 成功因素
1. 封面对比图效果直观，点击率高
2. 标题精准定位油皮人群，搜索流量大
3. 正文分步骤，易操作，收藏率高
4. 结尾提问引导，评论互动多

### 失败因素
1. 发布时间偏晚（22:30），错失黄金时段
2. 部分标签不够精准，流量分散
3. 没有及时回复评论，错失互动机会

### 改进方向
1. 下次选择 20:00-21:00 发布
2. 标签更精准，聚焦核心关键词
3. 发布后 1 小时内积极回复评论

### 可复用元素
1. 对比图封面效果好，继续使用
2. 痛点型+数字标题公式有效
3. 分步骤正文结构用户喜欢
```

## 更新 Obsidian 素材库

复盘完成后，更新 Obsidian 素材库：

1. **更新数据追踪**：记录各时间点的数据
2. **更新爆款公式**：将成功元素提炼为通用规律
3. **更新素材库**：将可复用的元素保存到素材库

**更新路径：**
- 数据追踪：`{vault}/12-小红书/数据追踪/{year}-{month}.md`
- 爆款公式：`{vault}/12-小红书/爆款公式/{element}.md`
- 素材库：`{vault}/12-小红书/素材库/{industry}/{title}.md`

## 半自动 vs 全自动

**半自动：**
- 同步前让用户确认内容包
- 发布后提醒用户记录数据
- 数据记录后一起分析复盘

**全自动：**
- 直接同步到草稿箱
- 自动生成复盘报告
- 自动更新 Obsidian 素材库

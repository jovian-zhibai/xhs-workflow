# 发布与复盘

笔记打磨完成后，使用 Wechatsync 自动同步到小红书草稿箱，发布后追踪数据表现。

## 同步发布

### 主方案：Wechatsync 自动同步（推荐）

**前置条件：**
1. 安装 Wechatsync CLI：`npm install -g @wechatsync/cli`
2. 安装 Chrome 扩展：搜索「文章同步助手」
3. 在 Chrome 中登录小红书
4. 配置 token：在 `config/xhs.json` 中设置 `wechatsync_token`

**AI 执行步骤：**

1. 从 `config/xhs.json` 读取 `wechatsync_token`
2. 设置环境变量：
   ```bash
   export WECHATSYNC_TOKEN="从配置文件读取的token"
   ```
3. 执行同步命令：
   ```bash
   # 同步到小红书草稿箱
   wechatsync sync output/xxx.md -p xiaohongshu -t "标题"
   ```
4. 确认同步成功，提示用户：
   ```
   ✅ 笔记已同步到小红书草稿箱

   📱 请按以下步骤完成发布：
   1. 打开小红书 APP
   2. 进入「我」→「草稿箱」
   3. 找到刚同步的笔记
   4. 检查封面图、文案、标签是否正常
   5. 选择合适时间点击发布

   ⏰ 建议发布时间：7-9 点、12-14 点、19-21 点
   ```

**故障排查：**

如果同步失败，按以下顺序检查：
1. **Token 无效**：检查 `config/xhs.json` 中的 `wechatsync_token` 是否正确
2. **Chrome 扩展未连接**：确保 Chrome 打开且「文章同步助手」扩展已启用
3. **未登录小红书**：在 Chrome 中打开小红书并登录
4. **网络问题**：检查代理配置

**备用方案：手动复制**

如果 Wechatsync 连接失败且无法修复，使用手动复制：
```bash
pbcopy < output/xxx.md
```

提示用户手动粘贴到小红书 APP。

### 发布后流程

1. 用户在 APP 中检查排版
2. 确认封面图、文案、标签是否正常
3. 选择合适时间点击发布

## 数据追踪

发布后记录数据表现，分析经验教训。

### 数据指标

**基础数据（必追）：**

| 指标 | 说明 | 重要性 |
|------|------|--------|
| 浏览量 | 笔记被多少人看到 | ⭐⭐⭐⭐⭐ |
| 点赞数 | 用户认可内容 | ⭐⭐⭐⭐⭐ |
| 收藏数 | 内容有实用价值 | ⭐⭐⭐⭐⭐ |
| 评论数 | 引发讨论/互动 | ⭐⭐⭐⭐⭐ |
| 分享数 | 社交裂变能力 | ⭐⭐⭐⭐ |
| 涨粉数 | 笔记带来的新粉丝 | ⭐⭐⭐⭐ |

**进阶指标（建议追）：**

| 指标 | 说明 | 为什么重要 |
|------|------|-----------|
| 点击率 | 曝光→点击的比例 | 衡量封面+标题吸引力 |
| 互动率 | (点赞+收藏+评论)/浏览 | 衡量内容质量 |
| 收藏率 | 收藏/浏览 | 干货笔记核心指标 |
| 评论质量 | 是否有深度评论 | 衡量内容触动程度 |

**归因分析（用于优化爆款公式）：**

| 指标 | 说明 | 优化方向 |
|------|------|---------|
| 封面类型 | 对比图/大字报/场景图 | 哪种封面点击率高 |
| 标题公式 | 数字型/痛点型/悬念型 | 哪种标题更吸引人 |
| 发布时间 | 7-9点/12-14点/20-21点 | 哪个时段效果好 |
| 标签效果 | 哪些标签带来了流量 | 优化标签策略 |

### 记录时间点

| 时间点 | 记录什么 |

| 1 小时 | 初步数据，判断是否限流 |

## 复盘

数据记录完成后，分析成功/失败因素，更新爆款公式。

### 复盘内容

**成功因素分析：**
- 哪些元素表现好？
- 为什么能吸引用户？
- 哪些标签带来了流量？
- 哪个时间段效果最好？

**失败因素分析：**
- 哪些元素表现差？
- 为什么没有吸引用户？
- 哪些标签没有效果？
- 是否被限流？为什么？

**改进方向：**
- 下次可以改进什么？
- 哪些元素可以复用？
- 哪些元素需要调整？

### 复盘报告格式

```markdown
## 复盘笔记

### 成功因素
1. 封面对比图效果直观，点击率高
2. 标题精准定位油皮人群，搜索流量大
3. 正文分步骤，易操作，收藏率高
4. 结尾提问引导，评论互动多

### 失败因素
1. 发布时间偏晚（22:30），错失黄金时段
2. 部分标签不够精准，流量分散
3. 没有及时回复评论，错失互动机会

### 改进方向
1. 下次选择 20:00-21:00 发布
2. 标签更精准，聚焦核心关键词
3. 发布后 1 小时内积极回复评论

### 可复用元素
1. 对比图封面效果好，继续使用
2. 痛点型+数字标题公式有效
3. 分步骤正文结构用户喜欢
```

## DraftPush 集成（可选）

如果用户安装了 Chrome 扩展 [DraftPush](https://github.com/jovian-zhibai/draftpush)，xhs-workflow 可以在输出到 `素材库/` 的同时，额外复制一份到待同步目录，触发 DraftPush 自动发布。

### 启用方式

在 `config/xhs.json` 中设置：

```json
{
  "draftpush": {
    "enabled": true,
    "sync_dir": "12-小红书/待同步"
  }
}
```

或在配置助手中说「是」即可自动开启。

### 输出结构

当 `draftpush.enabled` 为 `true` 时，额外输出到：

```
{vault}/12-小红书/待同步/{日期-标题简称}/
├── 笔记.md              # 含 YAML frontmatter
├── 笔记-cover.html      # 封面 HTML（保留）
├── cover.png            # 封面图片（DraftPush 用此文件上传）
└── 笔记-meta.json       # 元数据
```

### 封面图片生成

`cover.png` 按以下优先级自动生成：

| 优先级 | 方式 | 条件 | 命令/接口 |
|--------|------|------|-----------|
| 1 | Agnes 文生图 | `agnes_api_key` 已配置 | 调用 Agnes API |
| 2 | SenseNova 文生图 | `sensenova_api_key` 已配置 | 调用 SenseNova API（Agnes 失败时切换） |
| 3 | Chrome 无头截图 | Chrome 已安装 | `chrome --headless=new --screenshot=cover.png --window-size=1080,1440 file://.../cover.html` |
| 4 | 跳过 | 以上都不可用 | 日志提示用户手动截图 |

Chrome 路径自动检测：
```bash
CHROME=$(which google-chrome 2>/dev/null || echo "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome")
test -x "$CHROME" && echo "✅ Chrome 可用" || echo "⚠️ Chrome 不可用，跳过截图"
```

**YAML frontmatter 格式：**

```yaml
---
title: 笔记标题
tags: [标签1, 标签2, 标签3]
platforms: [xiaohongshu]
cover: cover.png
---
```

### 使用方式

1. 打开 Obsidian
2. 进入「12-小红书/待同步/」目录
3. DraftPush 扩展自动检测到待发布内容
4. 一键推送到小红书

## 更新 Obsidian 素材库

复盘完成后，更新 Obsidian 素材库：

1. **更新数据追踪**：记录各时间点的数据
2. **更新爆款公式**：将成功元素提炼为通用规律
3. **更新素材库**：将可复用的元素保存到素材库

**更新路径：**
- 数据追踪：`{vault}/12-小红书/数据追踪/{year}-{month}.md`
- 爆款公式：`{vault}/12-小红书/爆款公式/{element}.md`
- 素材库：`{vault}/12-小红书/素材库/{industry}/{title}.md`

## 半自动 vs 全自动

**半自动：**
- 复制前让用户确认内容包
- 发布后提醒用户记录数据
- 数据记录后一起分析复盘

**全自动：**
- 直接复制文案到剪贴板
- 自动生成复盘报告
- 自动更新 Obsidian 素材库

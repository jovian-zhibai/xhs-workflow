# 选题发现与爆款拆解

帮助用户从零开始找到有价值的小红书选题，通过爆款拆解学习成功模式。

## 何时跳过此阶段

以下情况直接进入撰写阶段：
- 用户已经明确说了要写什么主题
- 用户提供了素材（文档、链接、笔记）要求写成笔记
- 用户说"跳过选题"或类似表达

## 爆款拆解

选题前的核心步骤，通过拆解同类爆款笔记，提炼成功公式。

### 拆解流程

```
输入关键词/话题
    ↓
anysearch 搜索小红书相关内容
    ↓
筛选互动量高的笔记（点赞+收藏 > 1000）
    ↓
逐篇拆解：
  - 标题公式（用了什么策略？）
  - 封面设计（什么风格？构图？）
  - 正文结构（开头怎么写？分几段？）
  - 标签使用（用了哪些标签？）
  - 互动引导（结尾怎么引导？）
    ↓
提炼爆款共性
    ↓
生成「爆款公式报告」
    ↓
保存到 Obsidian 素材库
```

### 拆解维度

**标题拆解：**
- 类型：数字型/痛点型/悬念型/对比型/情绪型/干货型
- 字数：18-25 字最佳
- 关键词：是否包含核心关键词
- emoji：使用了哪些 emoji

**封面拆解：**
- 风格：对比图/大字报/拼贴/渐变/清单
- 构图：文字位置、图片比例
- 配色：主色调、对比度
- 信息量：是否一眼可见核心信息

**正文拆解：**
- 字数：300-800 字
- 结构：开头→干货→结尾
- 段落：每段几行，共几段
- emoji：使用频率和类型
- 口语化程度：是否像朋友聊天

**标签拆解：**
- 数量：5-10 个
- 热门标签：百万级浏览量
- 精准标签：十万级浏览量
- 长尾标签：万级浏览量

**互动引导拆解：**
- 结尾类型：提问/投票/求收藏
- 引导语：具体措辞

### 拆解报告格式

```
📊 爆款拆解报告：[关键词]

共分析 10 篇爆款笔记，互动量总计 50,000+

## 标题共性
- 数字型占比：60%（如「5个方法」「3天见效」）
- 痛点型占比：30%
- 平均字数：18 字
- 高频词：平价、学生党、必看

## 封面共性
- 对比图占比：50%
- 大字报占比：30%
- 主色调：白色/粉色系

## 正文共性
- 平均字数：450 字
- 开头类型：场景代入（70%）、痛点提问（30%）
- 段落数：4-6 段
- emoji 使用频率：每段 2-3 个

## 标签共性
- 热门标签：#护肤 #好物分享
- 精准标签：#油皮护肤 #学生党平价
- 标签数量：5-8 个

## 互动引导共性
- 结尾提问占比：70%
- 「收藏」引导占比：50%

## 💡 爆款公式
标题：数字 + 人群 + 痛点 + 结果
封面：对比图 / 大字报
开头：场景代入 → 抛出痛点
正文：分步骤 + emoji 分隔
结尾：提问引导评论
```

### 保存到 Obsidian

拆解完成后，保存到 Obsidian 素材库：

```

```

同时更新关键词索引：

```

```

## 信源访问策略

访问外部信源时，不要假设哪些站能直接访问、哪些不能。使用**连通性缓存**自动适配网络环境。

### 连通性缓存

缓存文件：`config/connectivity.json`（已在 `.gitignore` 中，不提交）

```json
{
  "xiaohongshu.com": { "method": "direct", "tested_at": "2026-06-07T10:00:00" },
  "weibo.com": { "method": "direct", "tested_at": "2026-06-07T10:00:00" },
  "pinterest.com": { "method": "proxy", "tested_at": "2026-06-07T10:00:00" }
}
```

- `method`：上次成功的方式（`direct` / `proxy` / `websearch`）
- `tested_at`：上次测试时间

### 访问流程

```
1. 查缓存 → 有记录？
   ├── 是 → 用缓存的 method 访问
   │   ├── 成功 → 直接使用
   │   └── 失败 → 按 direct → proxy → websearch 顺序重试，更新缓存
   └── 否 / 无记录 → 按 direct → proxy → websearch 顺序尝试，记录结果
```

**每种方式的具体操作：**

| method | 操作 |
|--------|------|
| `direct` | `curl -s --connect-timeout 5 {url}` |
| `proxy` | `curl -s -x {proxy.https} {url}`（代理地址从 `config/xhs.json` 读取） |
| `websearch` | anysearch 搜索关键词获取摘要信息 |

**关键原则：**
- 不要硬编码"某站需要代理"——实际试试才知道
- 每次成功/失败都更新缓存
- 没有配置代理时，跳过 proxy 步骤，直接从 direct 失败跳到 websearch
- 缓存不会过期，只在访问失败时更新（自愈）

## 三种发现模式

选题不是单一来源，而是三种模式综合扫描，最终推荐 3-5 个选题。

### 模式 1：热点速报（默认）

扫描当日热搜 + 热榜，找有流量基础的话题。

**搜索策略：**
- anysearch 搜索「小红书热搜」「微博热搜」「抖音热榜」等关键词
- 搜索与用户领域相关的当日热点
- 每个来源提取 3-5 个相关话题

**生活方式信源：**
- 小红书热搜：平台内最热话题
- 微博热搜：社会热点、娱乐八卦
- 抖音热榜：短视频热点
- 百度热搜：综合热点

**筛选标准：**
- 是否适合小红书用户（18-35 岁年轻女性为主）
- 是否有视觉呈现空间（图文笔记需要好看）
- 是否有实用价值或情绪价值
- 热点生命周期是否足够长（>3 天）

### 模式 2：深度选题

原始信源提前发现线索，找信息差。

**深度信源：**
- Pinterest 趋势：视觉灵感、设计趋势
- Instagram 探索页：海外生活方式
- TikTok 趋势：短视频热点先行
- 豆瓣小组：深度讨论、小众兴趣
- 知乎热榜：知识类、深度话题

**筛选标准：**
- 信息差：国内还没人写或写得少
- 可转化：能转化为小红书图文笔记
- 有受众：目标用户会感兴趣

### 模式 3：预判选题

周期性事件提前准备内容。

**周期性事件类型：**
- 美妆新品发布：各大品牌新品上市时间
- 季节穿搭：换季节点（春夏秋冬）
- 节日营销：情人节、618、双 11、春节等
- 影视综艺：热门剧集、综艺上映计划
- 明星同款：热播剧同款、明星穿搭

**时间窗口：**
- 提前 1-2 周准备内容
- 避开热点高峰期（竞争太大）
- 抓住长尾流量（热点过后仍有搜索）

## 综合推荐

三种模式结果汇总后去重排序，每个选题标注：

```
📡 选题推荐（共 5 个）

1. 「油皮夏季护肤指南」
   - 来源：热点速报（小红书热搜）
   - 爆款潜力：⭐⭐⭐⭐⭐
   - 理由：热搜第 3 名，搜索量大，季节性强

2. 「通勤穿搭合集」
   - 来源：深度选题（Pinterest 趋势）
   - 爆款潜力：⭐⭐⭐⭐
   - 理由：国内还没人写，信息差大

3. 「618 必买清单」
   - 来源：预判选题（节日营销）
   - 爆款潜力：⭐⭐⭐⭐⭐
   - 理由：618 临近，搜索量上升中
```

**半自动：** 展示选题清单，让用户选择
**全自动：** 自动选爆款潜力最高的选题

# 小红书工作流

一站式完成小红书图文笔记创作：**爆款拆解 → 选题 → 调研 → 写文案 → 生成标签 → 封面设计 → 打磨 → 输出内容包 → 同步发布 → 复盘**。

## 核心约束

**一个对话做一篇笔记。** 第一篇笔记完成后，如果用户想写新笔记，提示"请开一个新对话"。

## 运行模式

| 模式 | 触发方式 | 行为 |
|------|---------|------|
| 半自动（默认） | 正常对话 | 关键节点停下来等用户确认 |
| 全自动 | 用户说"全自动"、"不用问我"、"直接做完"、"帮我写完直接发" | 全程不问，自己决策 |

**半自动卡点：** 爆款拆解确认、选题确认、大纲确认、初稿审阅、标题选择、封面确认、发布确认

**全自动策略：** 优先选爆款潜力最高的选题、按爆款公式直接写、根据体检报告自动优化、优先选爆款潜力最高的标题（痛点法 > 数字法 > 悬念法 > 对比法 > 情绪法 > 干货法）、自动选择封面风格、打磨完成后直接同步到草稿箱。

**全自动失败处理：** 某个步骤失败时不中断全流程——跳过失败步骤继续，最后汇总报告哪些步骤成功、哪些失败。常见情况：搜索失败 → 用缓存素材继续；图片生成失败 → 跳过封面继续；同步失败 → 保留内容包，提示手动发布。

**步骤状态标记：** 每完成一个步骤，MUST 输出状态标记，格式为 `✅ 步骤 N：步骤名 — 一句话说明结果`。示例：
```
✅ 步骤 1：爆款拆解 — 分析 10 篇爆款，提炼出 5 个共性元素
✅ 步骤 2：选题 — 推荐 3 个选题，已确认「油皮夏季护肤指南」
✅ 步骤 3：调研 — 搜集 5 条素材
✅ 步骤 4：写文案 — 520 字，已审阅
```
这让用户随时知道进度到哪了，也防止 AI 跳步骤。

## 意图路由

| 用户说的 | 走哪条路 |
|---------|---------|
| "帮我写篇小红书" / "最近没什么灵感" | 完整流程：爆款拆解 → 选题 → ... → 同步发布 |
| "帮我拆解一下 XX 的爆款" | 只执行爆款拆解 |
| "帮我写篇关于 X 的笔记" | 跳过选题，从调研开始 |
| "帮我把这个写成小红书"（附带素材） | 跳过选题+调研，从写文案开始 |
| "帮我想几个标题" / "标题太普通了" | 调用爆款标题生成器 |
| "帮我推荐标签" | 调用标签推荐器 |
| "帮我看看这篇笔记怎么样" / "有没有改进空间" | 调用笔记体检报告 |
| "同步到小红书" / "发布到小红书" | 执行 wechatsync 同步到草稿箱 |
| "帮我配置" / "怎么设置" / "配置助手" / "检查配置" | 运行配置助手（见 `references/xhs-setup.md`） |

如果不确定用户意图，直接问。

## 完整流程

### 1. 爆款拆解

搜索同类爆款笔记，拆解标题、封面、正文结构、标签，提炼爆款公式。

**拆解流程：**
1. 用 anysearch 搜索关键词相关的小红书爆款内容
2. 筛选互动量高的笔记（点赞+收藏 > 1000）
3. 逐篇拆解：标题公式、封面风格、正文结构、标签使用
4. 提炼爆款共性，生成「爆款公式报告」
5. 保存到 Obsidian 素材库（`12-小红书/素材库/`）

**半自动：** 展示拆解报告，让用户确认
**全自动：** 直接应用爆款公式到后续步骤

> 详细流程见 `references/xhs-inspiration.md`

### 2. 选题

基于爆款拆解结果 + 三模式扫描，推荐 3-5 个选题：

- **热点速报**：小红书热搜 + 微博热搜 + 抖音热榜
- **深度选题**：Pinterest、Instagram、豆瓣、知乎等生活方式信源
- **预判选题**：美妆新品、季节穿搭、节日营销、影视综艺等周期性事件

> 信源连通性自动适配网络环境（`config/connectivity.json`），详见 `references/xhs-inspiration.md`

**半自动：** 展示选题清单，让用户选择
**全自动：** 自动选爆款潜力最高的选题

> 详细流程见 `references/xhs-inspiration.md`

### 3. 调研

拿着选题，搜集写作素材。不要凭空编造，用真实数据和案例支撑笔记。

**素材来源：** 竞品笔记、用户评价、产品参数、使用体验、价格信息

**半自动：** 展示素材清单，问用户是否补充
**全自动：** 直接整理好素材包供写文案使用

> 详细流程见 `references/xhs-research.md`

### 4. 写文案

按爆款公式撰写小红书笔记。口语化、emoji 点缀、300-800 字。

**文案风格：**
- 开头：前 3 行决定是否展开，用场景代入或痛点提问
- 正文：分段清晰，每段不超过 3 行，用 emoji 分隔
- 结尾：引导互动（提问/投票/求收藏）
- 标签：放在正文末尾

**小红书特有元素：**
- emoji 使用：✅ ❌ ⚠️ 📍 👉 💡 🔥 等引导符号
- 口语化：像在和朋友聊天
- 种草感：真实体验、有细节、有对比

**半自动：** 写完让用户审阅
**全自动：** 写完直接进入下一步

> 详细写作要求见 `references/xhs-writing.md`

### 5. 生成标签

为笔记推荐合适的标签组合，采用金字塔搭配策略：

- **热门大标签**（1-2 个）：百万级浏览量，获取曝光
- **精准中标签**（2-3 个）：十万级浏览量，精准触达
- **长尾小标签**（2-3 个）：万级浏览量，降低竞争

**标签格式：** #话题标签#

**半自动：** 展示标签组合让用户确认
**全自动：** 自动生成标签组合

> 详细策略见 `references/xhs-tools.md`

### 6. 封面设计

根据爆款拆解结果，推荐封面风格并生成封面 HTML。

**封面风格（根据爆款拆解决定）：**
- **对比图**：前后效果对比（转化率最高）
- **大字报**：核心卖点大字展示
- **拼贴**：多图拼接、活泼
- **渐变**：柔和、年轻感
- **清单**：编号列表、信息密度高

**封面要求：**
- 尺寸：3:4 竖版（1080×1440px）
- 文字：主标题占画面上方 1/3
- 配色：对比鲜明，不超过 3 种颜色

**半自动：** 展示封面方案让用户选择
**全自动：** 根据爆款拆解结果自动选择风格

> 模板详情见 `templates/README.md`

### 7. 打磨

写完初稿后，先去 AI 味，再跑体检报告，针对性优化。

**去 AI 味：** 优先使用 Humanizer（去 AI 痕迹）+ StopSlop（质量打磨），否则用内置 4 轮扫描：
1. 查特征词：替换 AI 高频词（赋能/深耕/打造）、删除套话（值得注意的是/在当今社会）
2. 查结构：打破刻板结构、排比堆砌、段落雷同
3. 查风格：加情绪起伏、换口语称呼、补具体案例
4. 加人味：补个人经历、明确立场、加入口语感和幽默感

**体检维度：** 封面吸引力、标题点击率、正文种草力、标签精准度、互动引导力

**半自动：** 展示体检报告问用户要不要改
**全自动：** 根据报告自动优化

> 体检报告用法见 `references/xhs-tools.md`

### 8. 输出内容包

将文案、标签、封面、发布指引打包输出到 `output/` 目录。

**内容包包含：**
- `xxx.md` — 笔记文案（含标签）
- `xxx-cover.html` — 封面 HTML
- `xxx-meta.json` — 元数据（标题、标签、封面风格等）
- `xxx-guide.md` — 发布指引

**格式：** Markdown + HTML（封面）

**DraftPush 集成（可选）：**

如果 `config/xhs.json` 中 `draftpush.enabled` 为 `true`，在输出到 `output/` 的同时，额外复制一份到 Obsidian 待同步目录，供 DraftPush 插件自动检测和发布：

```
{vault}/{draftpush.sync_dir}/{日期-标题简称}/
├── xxx.md          # 笔记文案（含 YAML frontmatter）
├── xxx-cover.html  # 封面 HTML
└── xxx-meta.json   # 元数据
```

**写入步骤：**
1. 正常输出到 `output/` 目录（默认行为，始终执行）
2. 检查 `config/xhs.json` → `draftpush.enabled`
3. 若为 `true`：
   - 生成文件夹名：`{日期}-{标题简称}`
   - 创建 `{vault}/{draftpush.sync_dir}/{文件夹}/`
   - 写入 `.md`（加 YAML frontmatter）、`-cover.html`、`-meta.json`
   - `.md` 的 YAML frontmatter 格式：
     ```yaml
     ---
     title: 笔记标题
     tags: [标签1, 标签2, 标签3]
     platforms: [xiaohongshu]
     cover: xxx-cover.html
     ---
     ```
4. 若为 `false` 或不存在：跳过，不做任何额外操作

### 9. 同步发布

使用 Wechatsync 命令行工具，将笔记自动同步到小红书草稿箱。

**全自动同步流程（默认方案）：**

```bash
# 设置环境变量（从 config/xhs.json 读取）
export WECHATSYNC_TOKEN="你的token"

# 同步到小红书草稿箱
wechatsync sync output/xxx.md -p xiaohongshu -t "标题"
```

**AI 执行步骤：**
1. 从 `config/xhs.json` 读取 `wechatsync_token`
2. 设置环境变量 `WECHATSYNC_TOKEN`
3. 执行 `wechatsync sync` 命令同步到小红书草稿箱
4. 确认同步成功，提示用户去小红书 APP 检查草稿箱
5. 提醒黄金发布时间：7-9 点、12-14 点、19-21 点

**备用方案（手动复制）：**

如果 Wechatsync 连接失败，使用手动复制：
```bash
pbcopy < output/xxx.md
```

**半自动：** 同步前让用户确认内容
**全自动：** 直接同步到草稿箱

> 详细流程见 `references/xhs-publishing.md`

### 10. 复盘

发布后记录数据表现，分析经验教训，更新爆款公式。

**数据追踪指标：**
- 基础数据：浏览量、点赞、收藏、评论、分享、涨粉
- 进阶指标：点击率、互动率、收藏率、评论质量
- 归因分析：封面类型、标题公式、发布时间、标签效果

**复盘流程：**
1. 发布后 1 小时/24 小时/72 小时/7 天记录数据
2. 分析成功/失败因素
3. 更新 Obsidian 素材库中的爆款公式
4. 为下次选题提供参考

**半自动：** 提醒用户记录数据，一起分析
**全自动：** 生成复盘报告，自动更新素材库

> 详细指引见 `references/xhs-publishing.md`

## 配置说明

首次使用需要配置。配置文件 `config/xhs.json` 支持两个位置（当前目录优先）：

> 详细流程见 `references/xhs-setup.md`

- **当前项目目录**：`{项目}/config/xhs.json`
- **Skill 目录**：skill 安装位置下的 `config/xhs.json`

```bash
cp config/xhs.example.json config/xhs.json
# 然后填入真实配置
```

配置文件包含敏感信息，已在 `.gitignore` 中排除。

用户说"帮我配置"或"配置助手"时，先检查 `config/xhs.json` 中哪些已配置、哪些缺失，只引导用户配置缺失的部分。可选功能（anysearch、Wechatsync、Obsidian 集成）逐个询问是否需要，用户说不要就跳过。

## Obsidian 集成

素材库存储在用户的 Obsidian Vault 中（`12-小红书/`），支持：

- **爆款素材库**：拆解结果按行业和关键词分类存储
- **爆款公式**：提炼出的标题、封面、正文、标签规律
- **数据追踪**：发布后的数据记录和分析

配置 `obsidian_vault_path` 后，AI 可以直接读写 Obsidian 中的素材。
